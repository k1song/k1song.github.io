1. 비판 개요

외부 분석 문서 "StockDigging 사업화의 현실"에 대한 항목별 검토이다. 해당 문서는 StockDigging의 사업화 가능성을 분석하며, 규제, 비용, 데이터, 시뮬레이션 등 다양한 측면에서 비판을 제기한다.

분류건수주요 항목
수용3건면책조항 미비, 시뮬레이션 지표 부족, 심리측정학적 검증 부재
반박4건카테고리 오류(로보어드바이저), 비용 과대산정, Exit 모델, 서비스 수준 과소평가
부분 수용2건데이터 출처/생존편향, 규제 리스크

2. 근본적 프레이밍 오류: 카테고리 문제

해당 비판 문서의 가장 큰 문제는 StockDigging을 "로보어드바이저"로 분류하고, 그 기준으로 평가한다는 점이다. 이 전제가 틀리면 비용 산정, 규제 요건, 경쟁사 비교, Exit 시나리오가 모두 틀어진다.

항목비판 문서의 전제StockDigging의 실제
서비스 분류로보어드바이저스크리닝 도구 + SaaS
규제투자일임업 (자기자본 5억)유사투자자문업 신고 (500만원)
경쟁사파운트, 핀트, 에임Simply Wall St, TipRanks, Seeking Alpha
필요 인력8~10명 (운용인력 포함)2~3명 (개발 + 운영)
1년차 비용13~16억원SaaS 모델 1~2억원 수준
Exit 모델금융사 매각ARR 기반 SaaS 밸류에이션
반박
StockDigging은 "이 종목을 사라"고 말하지 않는다. "이 기준으로 정렬하면 이런 종목이 나온다"를 보여주는 스크리닝 도구이다. Simply Wall St, Seeking Alpha, TipRanks 모두 종목 점수/순위를 보여주지만 투자일임업 등록 없이 전 세계에서 운영 중이다.

3. 수용하는 비판

3-1. 면책조항/법적 체계 미비

수용
비판: 면책조항이 2문장에 불과하다. 사업자 정보 미공개, 유사투자자문업 등록 여부 불명확.

이 비판은 타당하다. 현재 면책조항은 업계 기준에 미달하며, 서비스 공식 런칭 전 반드시 정비해야 한다.

대응 계획:

  • 면책조항 별도 페이지 작성 (투자 위험 고지, 과거 성과 ≠ 미래 수익 등)
  • 유사투자자문업 신고 진행 (자본금 500만원, 행정 절차)
  • 사업자 정보 공개 (서비스 하단 또는 별도 페이지)
  • 이용약관/개인정보처리방침 보강

시급도: 즉시. 런칭 전 완료 필수.

3-2. 시뮬레이션 지표 부족

수용
비판: 수익률 숫자만 제시하고 MDD, 샤프비율, 변동성 등 리스크 지표가 없다. 리밸런싱 기준 미공개.

이 비판도 타당하다. 수익률만으로는 전략의 품질을 판단할 수 없다.

대응 계획:

  • 시뮬레이션 결과에 MDD, 샤프비율, 변동성, 최대 연속 손실 추가
  • 리밸런싱 기준(ATR 기반) 투명하게 공개
  • 백테스트 한계(종가 기반, 슬리피지 미반영 등) 명시

시급도: 단기. 데이터는 이미 계산 가능하며 표시 기능 추가만 필요.

3-3. 심리측정학적 검증 부재

수용
비판: 30문항 퀴즈에 대한 학술적 검증(신뢰도, 타당도)이 없다. Riskalyze(노벨상 이론), FinaMetrica(52만건 데이터) 대비 근거 부족.

장기적으로 맞는 지적이다. 다만 모든 초기 서비스가 첫날부터 52만건 데이터를 가질 수는 없다.

대응 계획:

  • 사용자 데이터 축적 후 검사-재검사 신뢰도 분석 (장기)
  • 퀴즈 설계 근거 문서화 (각 문항이 어떤 스타일에 기여하는지 공개)
  • 학술적 검증은 PMF 달성 후 우선순위에 반영

시급도: 장기. 현 단계에서는 설계 근거 투명화로 대응.

4. 반박하는 비판

4-1. "투자일임업 등록 필요, 1년차 13~16억"

반박 — 카테고리 오류
비판의 전제: StockDigging이 투자일임업에 해당하므로 자기자본 5억, 인력 8~10명, 연간 13~16억이 필요하다.

StockDigging은 투자일임업이 아니다. 고객 자산을 운용하지 않고, 매매를 대행하지 않는다. 스코어링 기준으로 종목을 정렬하여 보여주는 스크리닝 도구이다.

글로벌에서 동일한 모델로 운영 중인 서비스들:

서비스기능투자일임업 등록
Simply Wall St종목 점수 + 순위 제시X
Seeking Alpha종목 등급 A+~F 제시X
TipRanksSmart Score 1~10 제시X
Finviz스크리닝 + 순위X

4-2. "경쟁사 = 파운트/핀트/에임"

반박 — 비교 대상 오류
비판: 로보어드바이저(파운트 AUM 1조, 핀트 AUM 4,389억)와 경쟁해야 하므로 진입 장벽이 높다.
항목로보어드바이저StockDigging
핵심 기능고객 자산 운용 (일임)종목 스코어링/정렬 (정보 제공)
고객 자산 보유O (수백~수천억)X
매매 대행OX
수익 모델운용 수수료 + 성과보수SaaS 구독 + B2B CPA
규제투자일임업유사투자자문업 (또는 미해당)
진입 자본5억원+500만원

4-3. "트래픽 모아서 매각은 환상"

반박 — 수익 모델 오해
비판: 금융사는 트래픽이 아닌 "라이선스 × 운용실적 × 전문인력"을 평가한다. Wealthfront처럼 14년이 걸린다.

StockDigging의 수익 모델은 "트래픽 모아서 금융사에 매각"이 아니다.

실제 수익 구조:

  • B2C: Premium 구독 (₩4,900/월)
  • B2B: 증권사 매매 연결 CPA (토스증권 등)
  • B2B: 스코어링 엔진 API 라이선스 (장기)

밸류에이션도 AUM 기반이 아닌 ARR(연간 반복 매출) 기반 SaaS 멀티플이 적용된다.

4-4. "출발선에 서기 전 단계"

반박 — 서비스 수준 과소평가
비판: 데이터 근거, 시뮬레이션 깊이, 학술적 기반, 법적 토대, 자본과 인력이 모두 부족하다.

현재 StockDigging이 달성한 것:

  • 라이브 서비스 운영 중 (stockdigging.com)
  • KR 2,663개 + US 6,716개 종목 커버
  • 109개 API 엔드포인트
  • 10,700회+ 시뮬레이션 수행
  • 6단계 체계적 연구 파이프라인 (ATR 최적화 ~ ML 전략 예측)
  • 크론 기반 자동 데이터 수집/갱신

"출발선 전"이라는 평가는 사업화 관점에서는 부분적으로 맞지만, 제품/기술 관점에서는 과소평가이다. 1인이 이 수준의 시스템을 구축한 것 자체가 실행력의 증거다.

5. 부분 수용하는 비판

5-1. 데이터 출처/생존편향

부분 수용
비판: 데이터 출처 미명시, 생존편향 미통제, 배당금/수수료 미반영.

수용: 데이터 출처를 서비스 내에 더 명확히 표시해야 한다. 생존편향 통제는 완벽하지 않다.

반박:

  • KR 데이터: DART API(공식), Naver Finance API — 출처 명시 가능
  • 배당금/수수료 미반영: 이미 서비스 내에 명시되어 있음
  • 생존편향: is_active 플래그로 상장폐지 종목 관리 중

5-2. 규제 리스크

부분 수용
비판: 종목을 보여주는 순간 규제 대상이다. 유사투자자문업 신고 필수.

수용: 유사투자자문업 신고는 안전을 위해 진행해야 한다.

반박: 투자일임업(자기자본 5억) 등록이 필요하다는 주장은 과도하다. StockDigging은 고객 자산을 운용하지 않으며, 매매를 대행하지 않는다.

6. 종합 대응 및 액션 플랜

이 비판 문서는 StockDigging을 로보어드바이저로 잘못 분류한 데서 출발하여, 규제/비용/경쟁사 분석이 전반적으로 과대 산정되었다. 그러나 법적 체계, 시뮬레이션 지표, 데이터 투명성에 대한 지적은 타당하며 즉시 대응이 필요하다.

이 문서의 존재 자체가 오히려 기회다. 미팅에서 선제적으로 언급하면:

  • "이런 비판이 있다. 우리도 인지하고 있다" → 객관성과 성숙도를 보여줌
  • "이건 맞다. 그래서 이렇게 대응한다" → 실행력을 보여줌

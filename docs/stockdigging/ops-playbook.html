1. 시스템 전체 구조

StockDigging v2는 v1의 단일 파이썬 서버에서 Next.js 프론트엔드 + FastAPI 백엔드 분리 구조로 전면 재설계되었다.

v1 vs v2 핵심 차이
  • v1: 단일 FastAPI + score_cache.pkl (P1 피팅/캘리브레이션 + 투자성향/대회 기능)

2. 배포 절차 (deploy.py)

코드 변경 후 배포는 로컬에서 python deploy.py 한 번으로 완료된다. GitHub Actions는 사용하지 않는다.

배포 전 필수 확인
  • git push 완료 확인: push 없이 deploy하면 서버는 구 버전을 배포함
  • 마이그레이션 파일 확인: backend/migrations/에 새 파일이 있으면 add_*.py(스키마)만 자동 실행됨. patch_*.py는 절대 포함하지 말 것 — 수십 분짜리 작업이 배포 중 서버를 다운시킴
  • 무거운 백그라운드 프로세스 확인: pgrep -fa precompute 로 실행 중인 작업 여부 확인

2-1. 배포 흐름 (Zero-downtime Atomic Swap)

  1. git pull + preflight 체크 (RAM / 디스크)
  2. backend 서비스만 stop (RAM 확보), frontend는 기존 .next로 계속 서빙
  3. .next → .next.old 백업 후 새 빌드 시작
  4. 빌드 실패 → .next.old 자동 복원, 다운타임 0
  5. 성공 시 services restart + HTTP 헬스체크
  6. 헬스체크 실패 → .next.old 자동 롤백
  7. 성공 시 .next.old 삭제, "✅ Deploy complete" 출력

2-2. 배포 후 확인

  • stockdigging.com 접속 — 랭킹 테이블 로딩 확인
  • 종목 상세 페이지 (/stock/005930 등) — 오늘 가격 표시 확인
  • 변경된 기능 직접 테스트 (TypeScript 컴파일 성공 ≠ 런타임 정상)
  • curl 헬스체크: curl http://127.0.0.1:8000/api/health → 200 확인
주의: "됐다고 하지 말고 실제로 확인"
  • 백엔드 API 200 응답 ≠ 프론트 정상
  • curl http://127.0.0.1:8000/ 로컬 테스트 ≠ nginx 통과 테스트
  • 배포 스크립트 "✅ Deploy complete" ≠ 기능 작동 — 반드시 브라우저에서 직접 확인

3. 일일 배치 상세 (run_daily.py)

크론이 호출하는 메인 배치 스크립트. 시장별로 데이터 수집 → precompute → IndexNow 핑 → 메일 알림 순서로 실행된다.

3-1. KR 배치 단계

단계함수설명
1_collect_kr_price()pykrx로 KR 종가 + 시총 수집, daily_prices upsert
2_collect_kr_valuation()DART 재무 + 종가로 PER/PBR/ROE/배당수익률 계산, valuations upsert
3_collect_index_prices()KOSPI/KOSDAQ/S&P500/NASDAQ/USD-KRW 지수 갱신
4_collect_macro_data()거시경제 지표 (금리, 환율 등)
5precompute_rankingsKR + ALL 정렬 JSON 재생성 (data/rankings/)
6precompute_stats필터 슬라이더용 stats_cache.json 재생성
7_ping_indexnow()Naver·Bing에 갱신 URL 알림 (즉시 색인 요청)
8notify()성공/실패 이메일 + Discord 발송

3-2. US 배치 단계

단계함수설명
1_collect_us_price()yfinance로 US 종가 수집, change_pct 계산 후 upsert
2_collect_index_prices()지수 갱신 (KR과 동일)
3_collect_macro_data()거시경제 지표
4~7precompute, stats, IndexNow, notifyKR과 동일
gap fill / --only-if-failed 동작 원리
  • gap fill (2번째 크론): --only-if-failed 없음 → 항상 실행. upsert 멱등성으로 이미 수집된 데이터는 덮어쓰고 누락 종목만 채움
  • retry (3·4번째 크론): --only-if-failed → batch_logs 테이블에서 오늘 price 단계 성공 기록이 있으면 스킵

3-3. DST 자동 처리 원리

US 크론을 고정 시각(20:10 / 21:10 UTC)에 두고, 스크립트 내부 _minutes_since_close()가 exchange_calendars를 통해 실제 마감 시각을 계산한다. 장 마감 전이면(minutes < 0) sys.exit(0)으로 자동 스킵 — 서머타임 수동 전환 불필요.

계절US 마감 (UTC)20:10 UTC 크론21:10 UTC 크론
EDT (3~11월)20:00마감 10분 후 → 실행 ✅마감 1h10m 후 → gap fill ✅
EST (11~3월)21:00장 중 → 자동 스킵 ⏭마감 10분 후 → main ✅

6. precompute 정적 캐시

매일 배치 성공 후 scripts/precompute_rankings.py가 모든 정렬 조합의 결과를 JSON으로 저장한다. 프론트엔드는 이 정적 파일을 nginx를 통해 직접 서빙받으므로 FastAPI 부하가 없다.

data/rankings/ ├── KR_market_cap.json # KR × 시가총액 정렬 ├── KR_per.json # KR × PER 정렬 ├── ... # 25개 sort × 3 market = ~75개 파일 ├── US_market_cap.json ├── ALL_market_cap.json # KR + US 합산 └── ALL_revenue.json
# 수동 재생성 (배치 실패 후 데이터 수집만 성공한 경우)
cd /var/www/stockdigging/backend

# KR만 재생성
/var/www/stockdigging/.venv/bin/python3 -m scripts.precompute_rankings --market KR

# US만 재생성
/var/www/stockdigging/.venv/bin/python3 -m scripts.precompute_rankings --market US

# ALL 재생성 (KR+US 합산 — --market US 로는 재생성 안 됨!)
/var/www/stockdigging/.venv/bin/python3 -m scripts.precompute_rankings --market ALL

# stats 캐시 재생성
/var/www/stockdigging/.venv/bin/python3 -m scripts.precompute_stats KR
/var/www/stockdigging/.venv/bin/python3 -m scripts.precompute_stats US
주의: ALL 파일은 별도 재생성 필요
--market US는 US_*.json만 재생성한다. ALL_*.json은 반드시 --market ALL을 따로 실행해야 갱신된다. 프론트엔드에서 ?market=ALL로 접속하면 ALL_*.json을 읽으므로 이 파일이 stale이면 데이터가 구버전으로 표시된다.
